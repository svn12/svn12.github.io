第三方生態整合的交鋒

開發者社群在配額限制下，主動串接第三方框架尋求解決方案

Omnigent Setup 支援
新增 Antigravity 作為一等公民，同時支援 headless 模式與 agy CLI 調用，使 SDK 與 claude/codex 同步對齊。
openclaw 驗證補丁
社群於 7 月初發布 PR 99733，整合 `agy --print` 與快取認證導入，用以加固因官方更新而破碎的 auth 邊界。
9router 代理路由
開發者積極嘗試在 agy 上游設置 API Gateway/Vertex 代理，期望藉由統一中轉繞過官方 CLI 的配額鎖定。

Pro/Free 帳號的奇特反差

近期開發者在 **Claude Code** 整合 Antigravity 時，遭遇了令人費解的 Bug：使用 **Google AI Pro** 付費帳號會持續認證失敗，但切換成 **Google Free** 免費帳號後卻能順暢運行。

「When using the Antigravity provider with Claude Code, requests consistently fail when authenticated with a Google Pro account. The exact same setup works correctly after switching to a Google Free account.」
— Github Issue decolua/9router #2225
⚠️
認證邊界異常阻斷
社群懷疑是 Google API 在回傳付費 Pro 帳號的權限標記 (Quota Tier) 時，與語言伺服器端的驗證規則發生衝突，反觀簡化版 Free 帳號沒有複雜的 Subscription 標籤而成功通行。

深入配額機制與社群炎上風暴

— 下一部分：解析「自動路由缺失」與「限額鎖死」的核心代碼爭議

Deliberate Design (刻意鎖定)

社群開發者在翻閱 `agy` 與官方 SDK 代碼後證實：這種「單一模型耗盡即鎖死全家」的懲罰機制並非漏洞，而是 Google 在協議端寫死的**商業限制設計**，主要用來限制高吞吐量帳號的調用頻率。

Single-Model 限額鎖死

開發者最痛恨的設計：一旦 **3.5-Flash** 的額度用盡，系統並不會降級或自動路由至其他可用模型，而是直接阻斷所有模型的運行。

「If you exhaust the quota for one model, it will be exhausted for all models. For example, if you exhaust the quota for the 3.5-Flash model, you won't be able to use the other models.」
— Github Issue google-gemini/gemini-cli #27858

Gemini CLI vs. Antigravity CLI 抉擇

Legacy Gemini CLI
  • Intelligent Auto-Edit - 具備流暢的代碼自動改寫體驗。
  • Auto Routing Mode - 模型額度耗盡時，會自動分流至備用模型。
  • 彈性第三方連線 - 可直接透過 LiteLLM 設定 API 代理。
Antigravity CLI (agy)
  • 單一配額鎖死 - 3.5-Flash 配額一乾，其餘模型全鎖。
  • 限額門檻極低 - 付費 Pro 帳號開一場 session 就耗盡額度。
  • 強硬認證綁定 - 不支援代理或 Vertex AI pay-as-you-go。
